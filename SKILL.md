# 金融报告风格蒸馏 Skill (Finance Report Style Distillation)

> **版本**: 1.0.0
> **类型**: 蒸馏工具 Skill（从内部报告中提炼写作风格，生成可复用的风格 Skill）
> **数据源**: 仅需内部报告文件（PDF/Word/Markdown/TXT）

***

## 元信息

- **Skill ID**: `finreport-distill-skill`
- **触发词**: `蒸馏报告风格`、`分析报告风格`、`生成报告Skill`、`distill report style`
- **适用场景**: 从已有的金融分析报告中提炼写作风格和分析框架，生成一个可复用的 Skill，让 AI 按照该风格自动撰写新报告
- **诚实边界**: 本 Skill 仅基于提供的报告进行风格提炼，无法补充报告之外的领域知识。提炼结果的置信度与报告数量和质量正相关。

***

## 核心理念

### 单源深潜策略

当只有内部报告这一种数据源时，我们采用\*\*"一份报告，六维扫描"\*\*策略：

一份报告虽然来源单一，但其中蕴含的信息是多维度的。我们通过 6 个分析维度对每份报告进行深度扫描，最大化信息提取量：

```
┌─────────────────────────────────────────────────┐
│              一份内部报告                          │
├──────────┬──────────┬──────────┬────────────────┤
│ 分析框架 │ 表达风格 │ 决策逻辑 │ 价值判断       │
│ 维度     │ 维度     │ 维度     │ 维度           │
├──────────┼──────────┴──────────┼────────────────┤
│ 报告结构 │ 风险偏好维度        │ 独特表达维度    │
│ 维度     │                     │                │
└──────────┴─────────────────────┴────────────────┘
         ↓ 六维扫描 ↓
┌─────────────────────────────────────────────────┐
│           风格 Skill 输出                        │
│  · 心智模型（分析框架）                           │
│  · 决策启发式（if-then 规则）                     │
│  · 表达 DNA（用词/句式/节奏/确定性）              │
│  · 诚实边界（能做/不能做）                        │
│  · 报告模板（结构/板块/格式）                     │
└─────────────────────────────────────────────────┘
```

### 三原则

1. **诚实优于完美**: 宁可 60 分诚实标注局限，也不要 90 分实际编造。单数据源场景下信息有限，必须明确标注哪些是报告原文证据，哪些是推理延伸。
2. **矛盾保留**: 报告中出现的观点不一致、前后矛盾不强制调和，而是保留为"内在张力"，标注矛盾类型和时间线。
3. **结构优于内容**: 风格蒸馏关注的是"怎么写"而非"写了什么"。提炼的是分析框架、表达习惯、决策模式，而非具体的投资观点或市场判断。

## 使用流程

### Phase 0: 初始化

当用户触发本 Skill 时，执行以下步骤：

**0.1 确认报告文件**

询问用户：

- "请提供需要分析的内部报告文件路径（支持 PDF/Word/Markdown/TXT 格式）"
- "这些报告属于哪个业务部门？覆盖哪些领域？"
- "您希望生成的 Skill 覆盖单一报告类型，还是综合多种类型？"

**0.2 创建工作目录**

在当前工作目录下创建：

```
finance-distill-{timestamp}/
├── input/              # 用户上传的报告原文
├── analysis/           # 六维分析结果
│   ├── 01_framework.md    # 分析框架维度
│   ├── 02_expression.md   # 表达风格维度
│   ├── 03_decision.md     # 决策逻辑维度
│   ├── 04_value.md        # 价值判断维度
│   ├── 05_structure.md    # 报告结构维度
│   └── 06_unique.md       # 独特表达维度
├── synthesis/          # 综合提炼结果
│   ├── mind-models.md     # 心智模型
│   ├── heuristics.md      # 决策启发式
│   ├── expression-dna.md  # 表达 DNA
│   ├── report-template.md # 报告模板
│   └── tensions.md        # 内在张力
└── output/             # 最终 Skill 输出
    └── SKILL.md           # 生成的风格 Skill
```

**0.3 报告预处理**

对每份报告执行：

1. 读取全文内容（使用 Read 工具）
2. 提取元信息：报告标题、日期、作者、类型、页数
3. 记录到 `input/report-index.md`

***

### Phase 1: 六维扫描分析

对每份报告，按以下 6 个维度逐一分析。每个维度生成独立的分析文件。

#### 维度 1: 分析框架 (`analysis/01_framework.md`)

**扫描目标**: 提取报告中反复使用的分析模型、理论框架、评估体系。

**提取要点**:

- 报告使用了哪些分析框架？（如：信用周期分析、DCF 估值、SWOT、波特五力等）
- 框架的应用方式是什么？（完整套用 / 适应性改造 / 多框架组合）
- 框架的输入要素有哪些？（需要哪些数据、指标、前提条件）
- 框架的输出形式是什么？（评级、评分、排名、定性判断）
- 不同报告类型是否使用不同框架？

**输出格式**:

```markdown
## 分析框架清单

### 框架 1: [框架名称]
- **出现频率**: 在 X/Y 份报告中使用
- **应用场景**: [什么类型的报告/什么情况下使用]
- **核心要素**: [输入要素列表]
- **输出形式**: [输出形式]
- **典型引用**: "[报告原文中的典型描述]"
- **置信度**: 高/中/低（基于出现频率和描述完整度）
```

#### 维度 2: 表达风格 (`analysis/02_expression.md`)

**扫描目标**: 提取报告的语言风格特征，量化表达习惯。

**提取要点**:

- **用词偏好**: 偏好哪些专业术语？是否有自创术语？
- **句式特征**: 长句/短句偏好？先结论后论证/先铺垫后结论？
- **确定性程度**: 使用"一定""必然"等强确定性词汇，还是"可能""或"等弱确定性词汇？
- **数字使用**: 精确数字 vs 范围表达？百分比 vs 倍数？
- **修辞手法**: 是否使用类比、比喻？频率如何？
- **称呼体系**: 如何称呼读者？（"投资者""委托人"等）
- **段落节奏**: 平均段落长度？信息密度？

**输出格式**:

```markdown
## 表达风格画像

### 用词偏好
- 高频术语: [术语列表 + 出现频率]
- 自创/特色术语: [术语列表 + 含义解释]
- 禁用/避免词汇: [词汇列表]

### 句式特征
- 典型句式 1: "[示例]"
- 典型句式 2: "[示例]"
- 段落结构: [描述]

### 确定性光谱
[极不确定] ←————→ [极确定]
报告位置: [标记在光谱上的位置]
典型表达: "[示例]"
```

#### 维度 3: 决策逻辑 (`analysis/03_decision.md`)

**扫描目标**: 提取报告中的推理链条和决策模式。

**提取要点**:

- **推理模式**: 演绎/归纳/类比/因果？
- **证据使用**: 引用数据的方式（精确引用/趋势描述/对比参照）
- **风险处理**: 如何讨论风险？（独立章节/融入分析/量化评估）
- **结论推导**: 从论据到结论的跳跃程度？（保守推导/大胆推断）
- **反事实思考**: 是否讨论"如果...会怎样"的场景？
- **时间维度**: 关注短期/中期/长期？时间跨度的选择标准？

**输出格式**:

```markdown
## 决策逻辑模式

### 推理链条模板
1. [前提条件]
2. → [分析步骤]
3. → [中间结论]
4. → [最终判断]
典型示例: "[报告原文]"

### 风险处理模式
- 风险提及位置: [章节位置]
- 风险量化方式: [描述]
- 风险与结论的关系: [描述]
```

#### 维度 4: 价值判断 (`analysis/04_value.md`)

**扫描目标**: 提取报告中的价值取向和判断标准。

**提取要点**:

- **分析优先级**: 风险优先 vs 收益优先？定量优先 vs 定性优先？
- **保守/激进程度**: 对不确定性的容忍度
- **客户导向**: 报告是为谁写的？如何平衡专业性和可读性？
- **独立性程度**: 是否敢于给出与市场共识不同的观点？
- **底线/红线**: 报告中绝对不会做的事或绝对不会说的话

**输出格式**:

```markdown
## 价值判断画像

### 分析优先级排序
1. [优先级 1]: [描述 + 证据]
2. [优先级 2]: [描述 + 证据]
...

### 保守/激进光谱
[极度保守] ←————→ [极度激进]
报告位置: [标记]
证据: "[原文引用]"
```

#### 维度 5: 报告结构 (`analysis/05_structure.md`)

**扫描目标**: 提取报告的标准化结构模板。

**提取要点**:

- **整体结构**: 报告由哪些板块组成？排列顺序？
- **必选板块**: 每份报告都包含的板块
- **可选板块**: 根据情况添加的板块
- **板块内部结构**: 每个板块内部的信息组织方式
- **格式规范**: 标题层级、数据展示方式、图表要求
- **开头/结尾模式**: 如何开头？如何结尾？

**输出格式**:

```markdown
## 报告结构模板

### 标准结构
1. [板块名称] - [内容说明] - [必选/可选]
2. [板块名称] - [内容说明] - [必选/可选]
...

### 板块详情: [板块名称]
- 内部结构: [描述]
- 典型长度: [字数/段落]
- 格式要求: [描述]
```

#### 维度 6: 独特表达 (`analysis/06_unique.md`)

**扫描目标**: 提取报告中的独特、个性化表达元素。

**提取要点**:

- **标志性语句**: 反复出现的特色表达
- **独特类比**: 报告中使用的生动类比或比喻
- **自创概念**: 分析师自己创造的术语或概念
- **幽默/讽刺**: 是否使用以及如何使用
- **情感色彩**: 报告整体的情感基调（冷静客观/热情推荐/谨慎警告）
- **口头禅/惯用语**: 高频出现的固定搭配

**输出格式**:

```markdown
## 独特表达清单

### 标志性语句
- "[语句 1]" - 出现 X 次
- "[语句 2]" - 出现 X 次

### 独特类比
- "[类比内容]" - 出处: [报告名称]

### 自创概念
- "[概念名称]": [含义解释]
```

### Phase 1.5: 覆盖度检查

完成六维扫描后，生成覆盖度矩阵：

```markdown
## 报告覆盖度矩阵

| 维度 | 信息充足度 | 关键发现数 | 缺失信息 | 置信度 |
|------|-----------|-----------|---------|--------|
| 分析框架 | ★★★★☆ | X 个 | [缺失项] | 高/中/低 |
| 表达风格 | ★★★☆☆ | X 个 | [缺失项] | 高/中/低 |
| 决策逻辑 | ★★★★★ | X 个 | [缺失项] | 高/中/低 |
| 价值判断 | ★★☆☆☆ | X 个 | [缺失项] | 高/中/低 |
| 报告结构 | ★★★★★ | X 个 | [缺失项] | 高/中/低 |
| 独特表达 | ★★★☆☆ | X 个 | [缺失项] | 高/中/低 |
```

**决策规则**:

- 如果任何维度置信度为"低"，向用户提示该维度信息不足，询问是否补充
- 如果 3 个以上维度置信度为"低"，建议用户增加报告数量后再重新蒸馏

### Phase 2: 结构化提炼

基于六维扫描结果，进行三重验证提炼。

#### 2.1 心智模型提炼 (`synthesis/mind-models.md`)

**三重验证标准**:

1. **跨报告复现**: 该分析框架/思维模式在至少 2 份报告中出现
2. **生成力**: 该模型可以用于分析新的标的/新的场景（而非仅适用于特定案例）
3. **排他性**: 该模型是该分析师/部门的特色，而非行业通用做法

**每个心智模型包含**:

```markdown
## 心智模型: [名称]

### 一句话描述
[用一句话概括这个思维模型的核心思想]

### 来源证据
- 报告 A: "[原文引用]"
- 报告 B: "[原文引用]"
- 复现次数: X/Y 份报告

### 应用方式
[如何将这个模型应用到新的分析任务中]

### 局限性
[这个模型在什么情况下不适用]

### 置信度
- 跨域复现: ✅/⚠️/❌
- 生成力: ✅/⚠️/❌
- 排他性: ✅/⚠️/❌
- 综合置信度: 高/中/低
```

**最低要求**: 至少提炼 3 个心智模型。如果不足 3 个，在诚实边界中标注。

#### 2.2 决策启发式提炼 (`synthesis/heuristics.md`)

从报告中提取可触发的 if-then 规则：

```markdown
## 决策启发式清单

### 启发式 1: [名称]
- **IF**: [触发条件]
- **THEN**: [应对策略]
- **来源**: [报告引用]
- **示例**: [具体案例]

### 启发式 2: [名称]
...
```

#### 2.3 表达 DNA 提炼 (`synthesis/expression-dna.md`)

```markdown
## 表达 DNA

### 称谓体系
- 自称: [如何自称]
- 读者称呼: [如何称呼读者]
- 第三方称呼: [如何提及分析对象]

### 句式 DNA
- 核心句式 1: "[模板]" - 使用频率: 高/中/低
- 核心句式 2: "[模板]" - 使用频率: 高/中/低

### 确定性表达光谱
- 强确定性词汇: [列表]
- 弱确定性词汇: [列表]
- 禁用确定性词汇: [列表]

### 数据呈现方式
- 精确引用格式: "[格式模板]"
- 趋势描述格式: "[格式模板]"

### 段落节奏
- 平均段落长度: [字数]
- 信息密度: 高/中/低
- 段落间过渡方式: [描述]
```

#### 2.4 报告模板提炼 (`synthesis/report-template.md`)

```markdown
3. **[板块 3]** (可选)

- 数据格式: [描述]
- 图表要求: [描述]
- 字数范围: [范围]

#### 开头模式
"[开头模板]"

"[结尾模板]"
```

### Phase 3: Skill 构建

基于提炼结果，生成最终的 SKILL.md 文件。生成的 Skill 结构如下：

```markdown
# [部门/分析师名称] 报告风格 Skill

> 由 finance-report-distill 自动生成
> 基于分析报告: [报告列表]
> 生成日期: [日期]

## 元信息
[基本信息]

## 心智模型
[从 synthesis/mind-models.md 导入]

## 决策启发式
[从 synthesis/heuristics.md 导入]

## 表达 DNA
[从 synthesis/expression-dna.md 导入]

## 报告模板
[从 synthesis/report-template.md 导入]

## 内在张力
[从 synthesis/tensions.md 导入]

## 诚实边界
- ✅ 本 Skill 基于以下报告提炼: [列表]
- ✅ 以下心智模型有充分证据支撑: [列表]
- ⚠️ 以下维度信息不足: [列表]
- ❌ 本 Skill 无法做到: [列表]

## 使用方式
当需要按照 [部门/分析师] 的风格撰写报告时：
1. 确定报告类型和标的
2. 选择对应的报告模板
3. 应用相关的心智模型进行分析
4. 按照表达 DNA 调整语言风格
5. 参考决策启发式处理风险和不确定性
```

### Phase 4: 质量自检

生成 Skill 后，执行以下自检：

| 检查项                | 标准       | 通过  |
| ------------------ | -------- | --- |
| 心智模型数量             | ≥ 3 个    | ✅/❌ |
| 每个心智模型有局限性说明       | 100%     | ✅/❌ |
| 表达 DNA 覆盖用词/句式/确定性 | 3/3      | ✅/❌ |
| 报告模板覆盖主要报告类型       | 全部       | ✅/❌ |
| 诚实边界已标注信息不足维度      | 有        | ✅/❌ |
| 内在张力已保留            | 有/无（可接受） | ✅/❌ |

## 增量进化

后续追加新报告时，支持增量更新：

### 追加报告

1. 将新报告放入 `input/` 目录
2. 重新执行 Phase 1 六维扫描（仅扫描新报告）
3. 合并到现有分析结果中
4. 重新执行 Phase 2 提炼
5. 更新 SKILL.md

### 对话纠正

用户可以直接对话修正生成的 Skill：

- "心智模型 X 的描述不准确，应该是..."
- "缺少一个重要的分析框架：..."
- "表达风格中应该增加..."

修正后更新对应的 synthesis 文件和 SKILL.md。

## Prompt 模板（独立使用版）

如果不想安装 Skill，可以直接使用以下 Prompt 模板。将 `[报告内容]` 替换为实际报告文本即可：

```
你是一位专业的金融分析报告风格分析师。请仔细阅读以下内部报告，从中提炼写作风格和分析框架。

## 报告内容
[报告内容]

## 分析要求

请从以下 6 个维度进行分析：

### 1. 分析框架
- 报告使用了哪些分析模型/理论框架？
- 框架的核心要素和输出形式是什么？

### 2. 表达风格
- 用词偏好（高频术语、特色术语、避免使用的词汇）
- 句式特征（长短句、结论位置）
- 确定性程度（强/弱确定性词汇使用习惯）

### 3. 决策逻辑
- 推理模式（演绎/归纳/类比/因果）
- 证据使用方式
- 风险处理方式

### 4. 价值判断
- 分析优先级排序
- 保守/激进程度
- 独立性程度

### 5. 报告结构
- 标准板块组成和排列顺序
- 必选/可选板块
- 格式规范

### 6. 独特表达
- 标志性语句
- 独特类比
- 自创概念

## 输出格式

请按以下格式输出分析结果：

1. **心智模型清单**（至少 3 个）
   每个模型包含：名称、一句话描述、来源证据、应用方式、局限性

2. **决策启发式清单**
   每条启发式包含：IF（触发条件）、THEN（应对策略）、来源

3. **表达 DNA**
   包含：称谓体系、句式 DNA、确定性光谱、数据呈现方式

4. **报告模板**
   包含：标准结构、各板块要求、格式规范

5. **诚实边界**
   标注：能做什么、不能做什么、信息不足的维度
```

## 附录：报告类型参考

以下为常见的金融分析报告类型，用于辅助分类：

| 类型   | 说明             | 典型结构                             |
| ---- | -------------- | -------------------------------- |
| 信用研究 | 信用主体跟踪、信用风险评估  | 主体概况 → 业务分析 → 财务分析 → 信用评价 → 风险提示 |
| 权益研究 | 公司研究、行业研究、估值分析 | 公司概况 → 行业分析 → 财务分析 → 估值模型 → 投资建议 |
| 宏观研究 | 宏观经济分析、政策影响    | 宏观数据 → 趋势判断 → 政策分析 → 影响评估 → 配置建议 |
| 固收研究 | 利率分析、债券策略      | 市场回顾 → 利率分析 → 供需分析 → 策略建议        |
| 策略报告 | 市场展望、投资策略      | 市场回顾 → 核心观点 → 配置建议 → 风险提示        |
| 专题研究 | 特定事件/主题深度分析    | 事件概述 → 影响分析 → 历史对比 → 展望判断        |
| 评级报告 | 信用评级/主体评级      | 评级结论 → 主体分析 → 支撑因素 → 压力测试 → 评级展望 |
