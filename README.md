# 金融报告风格蒸馏 Skill (Finance Report Style Distillation)

> 从特定金融分析报告中提炼写作风格，生成可复用的风格 Skill，让 AI 按照特定风格自动撰写新报告。

## 快速开始

### 最低要求
- 3-10 份同一部门/分析师的内部报告（PDF/Word/Markdown 格式）
- Claude Code 环境（推荐）或任意 AI 对话工具

### 使用步骤

1. **安装 Skill**：将本文件夹复制到 Claude Code 的 skills 目录下
2. **触发蒸馏**：在对话中输入 `蒸馏报告风格`
3. **提供报告**：上传报告文件路径
4. **确认结果**：查看蒸馏摘要并确认
5. **使用生成物**：用生成的风格 Skill 撰写新报告

## 核心设计

### 单源深潜策略

仅依赖内部报告一种数据源，通过 **"一份报告，六维扫描"** 最大化信息提取：

| 维度 | 扫描目标 | 产出 |
|------|---------|------|
| 分析框架 | 反复使用的分析模型、理论框架 | 心智模型（≥3个） |
| 表达风格 | 用词、句式、确定性、节奏 | 表达 DNA |
| 决策逻辑 | 推理链条、证据使用、风险处理 | 决策启发式 |
| 价值判断 | 优先级、保守程度、独立性 | 价值画像 |
| 报告结构 | 板块组成、格式规范 | 报告模板 |
| 独特表达 | 标志性语句、类比、自创概念 | 独特表达清单 |

### 三大原则

1. **诚实优于完美** — 明确标注信息不足的维度，不编造
2. **矛盾保留** — 报告中的观点差异保留为"内在张力"，不强制调和
3. **结构优于内容** — 只蒸馏"怎么写"，不蒸馏"写了什么"

### 质量门控

- **Phase 1.5**：覆盖度矩阵检查（6维度信息充足度评估）
- **Phase 2.5**：提炼结果预览确认
- **Phase 4**：6项自动化质量自检

## 蒸馏产出物

| 产出 | 说明 |
|------|------|
| 心智模型 | 核心分析框架（含名称/描述/证据/应用/局限） |
| 决策启发式 | IF-THEN 规则集 |
| 表达 DNA | 用词/句式/确定性/节奏的量化画像 |
| 报告模板 | 各类报告的结构/格式/开头结尾模式 |
| 内在张力 | 报告中的矛盾观点（不强制调和） |
| 诚实边界 | 能做/不能做的明确声明 |

## 报告数量建议

| 数量 | 效果 | 场景 |
|------|------|------|
| 1-2 份 | 基础风格描绘，置信度低 | 快速原型验证 |
| 3-5 份 | 核心特征提取，置信度中 | **最低推荐** |
| 5-10 份 | 稳定风格画像，置信度高 | 推荐 |
| >10 份 | 精细风格画像，可按类型分别蒸馏 | 理想 |

## 增量进化

- **追加报告**：放入新报告，重新扫描提炼，更新 Skill
- **对话纠正**：直接对话修正（如"心智模型 X 描述不准确"）

## 技术来源

本 Skill 融合了三套开源 Skill 框架的设计模式：

- **毛选 Skill** — 心智模型结构化、表达 DNA 量化、诚实边界
- **女娲 Skill** — 三重验证提炼法、质量门控、矛盾保留
- **同事 Skill** — 预览确认、增量进化、标准化目录结构

针对单数据源场景做了简化：去掉多 Agent 并行采集、多源适配器、入口分流等复杂机制。
